Administrative Client

Startup

Start the client by typing, from the bin subdirectory of the install directory, "./cli" for mac or linux and "./cli.bat" for cygwin and "cli.bat" for Windows.

Plain Client

If you are having problems with the colors, text positiong or windows resizing, you may wan to use the plain client. To run the plain client, add "--plain" to the cli command.

Use a Cluster

You must use a cluster in order to operate on in. Type "use cluster <cluster name>". The cluster name must match a config file in the config directory. The name of the config file must follow "config-<cluster name>.json".

Deploy Cluster

To deploy the image to the servers in the cluster, type "deploy cluster". Note that you must setup passwordless ssh between all the servers and between all the servers and the client. Deploy cluster is not supported for Windows.

Start Cluster

To start a cluster type "start cluster".

Start Server

To start a single server type "start server <shard> <replica>". Where shard and replica indicate which server to start.

Stop Cluster

To stop the cluster type "stop cluster".

Stop Server

To stop a single server type "stop server <shard> <replica>". Where shard and replica indicate which server to stop.

Purge Cluster

To purge a cluster type "purge cluster". This will purge and stop the cluster.

Create a Database

You must create a database in order to insert data. Type "create database <database name>".

Use a Database

To use a database type "use database <database name>" You must use a database in order to operate on it.

Reconfigure Cluster

To reconfigure the cluster based on the current config file, type "reconfigure cluster". This needs to be done, for example, if you change the addresses of your servers.

Reload Server

To reload a server from a peer, type "reload server <shard> <replica>, where shard and replica specify the server you wish to reload. You would need to take this action in the case you are replacing a bad server or disk where the database resides.

Reload Server Status

To get the status of a server reload, type "reload server status <shard> <replica>, where shard and replica specify the server you are reloading. Returns "complete" or "running" depending on the state of the reload.

Reload Replica

To reload a replica from its peers, type "reload replica <replica>, where replica specifies the replica you wish to reload. You would need to take this action in the case you are adding a new replica.

Reload Replica Status

To get the status of a replica reload, type "reload replica status <replica>, where replica specifies the replica you are reloading. Returns "complete" or "running" depending on the state of the reload.
