Administrative Client

Commandline Mode

You can use the client in a non-interactive mode. The commandline options are:
  • --cluster=[cluster name]
  • --db=[db name]
  • --command=[command]

Interactive Mode

Start the interactive client by typing, from the bin subdirectory of the install directory, "./cli" for mac or linux and "./cli.bat" for cygwin and "cli.bat" for Windows.

Plain Client

If you are having problems with the colors, text positiong or windows resizing, you may wan to use the plain client. To run the plain client, add "--plain" to the cli command.

Run a Script

A command script consists of a list of commands separated by a newline. You can run a script of commands by entering the command "run script [script name]". Script name is either the fully qualified path name or it is just the filename if the file is located in the sonicbase/scripts directory.

You can run a script from the comand line using the following option: "--script=[script name]"

Start Cluster

To start a cluster type "start cluster".

Start Server

To start a single server type "start server <shard> <replica>". Where shard and replica indicate which server to start.

Start Server and Disable it

To start a single server that is disabled type "start server <shard> <replica> disable". Where shard and replica indicate which server to start. You may want to start a server disabled if the server is being replaced and has no data on it. Follow this command with the "reload server" command.

Stop Cluster

To stop the cluster type "stop cluster".

Stop Server

To stop a single server type "stop server <shard> <replica>". Where shard and replica indicate which server to stop.

Purge Cluster

To purge a cluster type "purge cluster". This will purge and stop the cluster.

Create a Database

You must create a database in order to insert data. Type "create database <database name>".

Use a Database

To use a database type "use database <database name>" You must use a database in order to operate on it.

Restart Cluster

To do a rolling restart of the cluster, type "restart cluster". This command is only available in the Pro version.

Disable Server

To disable a server, type "disable server [shard] [replica]. This will prevent it from accepting further commands.

Enable Server

To enable a server, type "enable server [shard] [replica]. This will allow a server to start accepting commands.

Purge Install

To purge the install directory across the cluster, type "purge install".

Quit or Exit

To exit the client, type "quit" or type "exit".

Healthcheck

To run a healthcheck across the cluster, type "healthcheck".

Clear

To clear the screen, type "clear".

Build Config

For non-aws config building:
Create a text file listing the address of each server you want in the cluster. Type the following command:
  build config [path to text file]

For aws-based config bulding:
See AWS Integration.

Reconfigure Cluster

To reconfigure the cluster based on the current config file, type "reconfigure cluster". This needs to be done, for example, if you change the addresses of your servers.
